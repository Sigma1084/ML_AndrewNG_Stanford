<h2 id="k-means-clustering">K-Means Clustering</h2>
<p>(The algorithm is an iterative algorithm)<br>
Suppose we need to cluster a set of points into k clusters.</p>
<p>We randomly define k points to be centroids.<br>
Then, Iterate {</p>
<ul>
<li>Group the set into k groups, such that for each centroid, each group containing points whose closest centroid is that one</li>
<li>Calculate the centroid of k groups and relocate the centroids</li>
</ul>
<p>}</p>

<h3 id="more-on-k-means">More on K-means</h3>
<ul>
<li>Choosing k random points to be k centroids is widely used and accepted</li>
<li>The algorithm might reach a local minima and give a wrong answer. One way to counter that is to run the clustering a set number of times (say 100) and choosing the clustering with the minimal Distortion</li>
</ul>
<blockquote>
<p><strong>Choosing the value of k</strong>: Elbow Method (plotting cost vs k) can be used and the value where slope gets lower can be chosen. Often times, it’s hard to determine and hence no high expectations. Better to ask for what purpose we are performing k-means</p>
</blockquote>
